Releaseinformation
 Microsoft JDBC Driver 6.0 för SQL Server
 Maj 2016

------------
INTRODUKTION
------------
Den här filen innehåller den senaste eller annan viktig information för Microsoft JDBC
 Driver 6.0 för SQL Server-dokumentationen. Du bör läsa den här filen helt innan du installerar
 JDBC-drivrutinen.

Din återkoppling är mycket viktig för oss och vi försöker besvara din feedback så fort
 vi kan. Information om hur du lämnar feedback genom att använda JDBC-drivrutinens bloggar och onlineforum finns på 
 Microsoft JDBC Driver 6.0 för SQL Server-sidan på http://msdn.microsoft.com/data/jdbc.

INSTALLATION
------------
Instruktioner för hur du installerar JDBC-drivrutinen finns i install.txt.  Se den filen för 
 information om hur du installerar JDBC-drivrutinen på Windows och Unix-operativsystem.

  Windows Server 2008 SP2

Listan ovan är ett exempel på vissa av de operativsystem som stöds. JDBC-drivrutinen är utformad
 att fungera med alla operativsystem som stöder användningen av en virtuell Java-maskin (JVM). Däremot har endast operativsystem
 Oracle Solaris (x86), SUSE Linux och Windows Vista Service Pack 2 eller senare
 testats.

JAR-filer:
---------
För att stödja bakåtkompatibilitet och möjliga uppgraderingsscenarier omfattar JDBC Driver 6.0
 fyra JAR-klassbibliotek i varje installationspaket: sqljdbc.jar, sqljdbc4.jar, sqljdbc41.jar
 och sqljdbc42.jar.  

Viktig information: sqljdbc.jar och sqljdbc4.jar tillhandahålls bara för bakåtkompatibilitet och innehåller
 inte de nya funktioner som ingår i denna version.  Endast sqljdbc41.jar och sqljdbc42.jar
 innehåller de nya funktionerna.

ÄNDRINGSLISTA
-----------
Ändringar från 4.2 till 6.0:

Ny funktion: Always Encrypted – Programutvecklare kan nu använda Always Encrypted (AE)-funktionen som är
 tillgänglig sedan SQL Server 2016 CTP (eller senare). JDBC-drivrutinen möjliggör Always Encrypted-funktionen genom att transparent
 kryptera och dekryptera känsliga data lagrade i krypterade databaskolumner i SQL Server. Mer information
 finns i MSDN-dokumentationen om användning av Always Encrypted med JDBC-drivrutinen. 

Ny funktion: IDN (Internationalized Domain Name) – JDBC-drivrutinen stöder nu IDN-namn för
 server- och instansnamn. Mer information finns i MSDN-dokumentationen om hur du använder IDN med JDBC-drivrutinen.

Ny funktion: Tabellvärdeparametrar – de här parametrarna tillhandahåller ett enkelt sätt att ordna flera rader med data från
 ett klientprogram till SQL Server utan att kräva flera rundturer eller speciell logik på serversidan för bearbetning av data. 
Du kan använda tabellvärdeparametrar för att kapsla in rader med data i ett klientprogram och skicka data till servern i ett
 kommando med parametrar. Mer information finns i MSDN-dokumentationen om tabellvärdeparametrar.

Ny funktion: AAD (Azure Active Directory) – AAD-autentisering är en mekanism för anslutning till Azure SQL Database v12 med
 identiteter i AAD. Använd AAD-autentisering för central hantering av identiteter för databasanvändare och som ett alternativ till SQL
 Server-autentisering. Med JDBC Driver 6.0 kan du ange dina AAD-inloggningsuppgifter i JDBC-anslutningssträngen
 för att ansluta till Azure SQL DB. Mer information finns i MSDN-dokumentationen om inställning av anslutningsegenskaper för JDBC-drivrutinen.

Förbättring: AlwaysOn Availability Groups (AG) – Drivrutinen stöder nu transparenta anslutningar till AlwaysOn Availability
 Groups. Drivrutinen upptäcker snabbt den aktuella AlwaysOn-topologin för din serverinfrastruktur och ansluter till den
 aktuella aktiva servern transparent. Mer information finns i MSDN-dokumentationen om inställning av anslutningsegenskaper för
 JDBC-drivrutinen.

Förbättring: JDBC-drivrutinen stöder nu hämtning av parametermetadata med förberedda satser för komplexa frågor
 t.ex. underfrågor/föreningar. Observera att denna förbättring bara är tillgänglig när du använder SQL Server 2012 och nyare versioner.


------------
KÄNDA PROBLEM
------------
Följande är kända problem med Microsoft JDBC Driver 6.0 för SQL Server:

1) PARAMETERMETADATABEGRÄNSNINGAR VID ANVÄNDNING AV SQL SERVER 2008 ELLER 2008 R2

1.a) PreparedStatement.getParameterMetadata() kastar ett SQLException vid användning
 med en MERGE-fråga som innehåller parametrar
1.b) Parametermetadata för förberedda satser fungerar inte med frågor som har underfrågor/föreningar.


2) JavaKerberos och virtuella IBM-maskiner

Om du kör i Windows-miljöer kan du behöva ange principalnamnet i
 konfigurationsfilen för att få Java Kerberos att fungera Ett exempel: 
SQLJDBCDriver {
  com.ibm.security.auth.module.Krb5LoginModule krävs
  seDefaultCcache=true
  credsType=initiator
  principal=principalname@principaldomain;
};


3) ANSLUTNINGEN KOPPLAS NED MED SQL AZURE

Vid anslutning till SQL Azure kan oanvända anslutningar avbrytas av en nätverkskomponent (t.ex. en
 brandvägg) efter en period av inaktivitet. För att undvika att tappa oanvända anslutningar av en nätverkskomponent
 ska följande registerinställningar (eller deras icke-Windows-motsvarigheter) anges på operativ
system där drivrutinen är inläst:
           
Registerinställningar                                                                   Rekommenderat värde
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\KeepAliveTime            30000
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\KeepAliveInterval         1000
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\TcpMaxDataRetransmission  10
